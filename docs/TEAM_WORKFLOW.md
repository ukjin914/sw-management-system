# 팀 협업 워크플로우 (상세)

> 이 문서는 [CLAUDE.md](../CLAUDE.md)의 협업 가이드를 보다 상세하게 풀어놓은 작업 매뉴얼입니다.
> 사용자/codex/Claude 간 보고 라인과 단계별 산출물 형식을 정의합니다.

---

## 팀 구성 (4팀 + 1검증자)

### 🧭 기획팀
- **역할:** 사용자 요구사항을 수집·정리해 1차 기획서를 작성한다.
- **산출물:** `docs/plans/{기능명}.md`
- **포함 항목:**
  - 배경/목표
  - 사용자 시나리오
  - 화면/UI 흐름
  - 기능 요건 (FR)
  - 비기능 요건 (NFR)
  - DB팀 자문 결과
  - 우려사항/의사결정 필요 항목

### 🗄️ 데이터베이스팀 (자문 역할)
- **역할:** 기획 단계에서 스키마·인덱스·마이그레이션 영향을 검토해 의견을 낸다.
- **산출물:** 기획서 내 "DB팀 자문" 섹션 또는 별도 `docs/db-impact/{기능명}.md`
- **포함 항목:**
  - 영향받는 테이블/컬럼
  - 마이그레이션 필요 여부 (V###_xxx.sql)
  - 인덱스/제약 변경
  - 데이터 정합성 리스크

### 🛠️ 개발팀
- **역할:** 승인된 기획서를 바탕으로 개발계획서를 작성하고, 사용자 최종승인 후 코드를 작성한다.
- **산출물:** `docs/dev-plans/{기능명}.md` + 실제 코드 변경
- **포함 항목:**
  - 영향 범위 (Controller/Service/Repository/Entity/Template)
  - 변경 파일 목록과 변경 내용 요약
  - 단계별 작업 순서
  - 회귀 테스트 항목
  - 롤백 전략

### 🧪 테스트 및 검증팀 (= codex)
- **역할:** 작성된 코드의 정합성·동작·회귀를 검증한다.
- **Claude는 이 단계를 직접 수행하지 않는다.**
- **산출물:** codex가 사용자에게 직접 보고하는 검증 결과

### 🤖 codex (검증자/게이트웨이)
- **역할:** 모든 산출물을 검토하고 사용자에게 결과를 전달한다.
- **호출 방식:** Claude가 codex CLI를 호출 → codex 응답을 사용자에게 전달

## codex CLI 통합

### 설치/설정 가정
codex CLI는 사용자 환경에 이미 설치되어 있다고 가정한다.

### 검토 요청 명령 패턴

```bash
# 기획서 검토
codex review --type plan --file docs/plans/feature-xxx.md

# 개발계획서 검토
codex review --type dev-plan --file docs/dev-plans/feature-xxx.md

# 코드 검증 (특정 파일들)
codex verify --files \
  src/main/java/com/swmanager/system/service/XxxService.java \
  src/main/java/com/swmanager/system/controller/XxxController.java

# 테스트 실행 위임
codex test --suite inspect-report
```

### 응답 처리

codex의 응답을 받아 다음 형식으로 사용자에게 전달:

```markdown
## [codex 검토 결과] {산출물명}

**평가:** ✅ 승인 / ⚠ 수정필요 / ❌ 반려

**주요 의견:**
- ...

**권장 수정사항:**
1. ...

---
다음 단계: {반영 / 최종승인 대기}
```

## 예외 처리 (Fast-track)

다음 경우는 위 절차를 생략할 수 있다 (사용자 판단/지시):

- 명백한 오타·버그 수정
- 단일 파일의 텍스트/스타일 미세 조정
- 사용자가 직접 "바로 수정"이라 명시한 경우
- 긴급 hotfix

이 경우에도 사용자에게 **간단한 변경 요약**은 반드시 전달한다.

## 자동화 후크

### "작업완료" → 자동 커밋+푸시
사용자가 "작업완료"라고 말하면:
1. `git status` / `git diff --stat`로 변경 확인
2. 관련 파일만 `git add`로 명시적 스테이징 (민감/불필요 파일 제외)
3. 변경 의도 중심의 커밋 메시지로 `git commit`
4. `git push`

### 서버 재시작 자동화
서버 재시작이 필요한 코드/리소스 변경 후 즉시:
```bash
bash server-restart.sh
```

별도 승인 요청 없이 수행한다.
